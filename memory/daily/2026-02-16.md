## OpenClaw Streaming Architecture (Feb 16, 12:08 PM)

**Source:** https://docs.openclaw.ai/concepts/streaming

### Key Concepts Learned

#### 1. Two Streaming Layers

**Block Streaming (Channel Messages):**
- Emits completed blocks as assistant writes
- Normal channel messages (not token deltas)
- Coarse chunks, not real-time tokens

**Token-ish Streaming (Telegram Only):**
- Updates temporary preview message with partial text
- Only Telegram supports this live preview

#### 2. Block Streaming Controls

**Configuration:**
```yaml
agents.defaults.blockStreamingDefault: "on" | "off"  # default: off
agents.defaults.blockStreamingBreak: "text_end" | "message_end"
agents.defaults.blockStreamingChunk:
  minChars: 100
  maxChars: 2000
  breakPreference: "paragraph" | "newline" | "sentence"
```

**Break Modes:**
- `text_end`: Stream blocks as soon as chunker emits (real-time)
- `message_end`: Wait until message finishes, then flush (batch)

#### 3. Chunking Algorithm

**EmbeddedBlockChunker:**
- Low bound: Don't emit until buffer >= minChars
- High bound: Prefer splits before maxChars
- Break preference: paragraph ‚Üí newline ‚Üí sentence ‚Üí whitespace ‚Üí hard break
- Code fences: Never split inside, close/reopen if forced

#### 4. Coalescing (Merge Streamed Blocks)

**Purpose:** Reduce "single-line spam" while keeping progressive output

**How it works:**
- Waits for idle gaps (idleMs) before flushing
- Caps at maxChars
- minChars prevents tiny fragments
- Joiner based on breakPreference

**Defaults:**
- Signal/Slack/Discord: minChars 1500 (higher to reduce noise)

#### 5. Human-Like Pacing

**Config:**
```yaml
agents.defaults.humanDelay: "off" | "natural" | "custom"
# natural: 800-2500ms randomized pause between blocks
# custom: minMs/maxMs
```

**Purpose:** Makes multi-bubble responses feel more natural

#### 6. Telegram Preview Streaming (Special)

**Modes:**
- `partial`: Preview updates with latest stream text
- `block`: Preview updates in chunked blocks
- `off`: No preview streaming

**Process:**
```
sendMessage (temporary preview)
  ‚Üì
editMessageText (updates during generation)
  ‚Üì
final text-only reply ‚Üí edit in place
  ‚Üì
complex/media ‚Üí cleanup preview + normal delivery
```

**Note:**
- Preview streaming separate from block streaming
- If block streaming enabled for Telegram, preview skipped (avoid double)
- /reasoning stream writes into live preview (Telegram only)

#### 7. Channel-Specific Behavior

**Non-Telegram Channels:**
- Block streaming OFF unless explicitly enabled
- Need `*.blockStreaming: true` per channel

**Telegram:**
- Can have preview streaming WITHOUT block replies
- Block replies are separate setting

### Configuration Summary

```yaml
agents:
  defaults:
    blockStreamingDefault: "on"
    blockStreamingBreak: "text_end"  # or "message_end"
    blockStreamingChunk:
      minChars: 100
      maxChars: 2000
      breakPreference: "paragraph"
    blockStreamingCoalesce:
      minChars: 1500  # higher for Discord/Signal/Slack
      maxChars: 4000
      idleMs: 500
    humanDelay: "natural"  # 800-2500ms

channels:
  telegram:
    streamMode: "partial"  # or "block" | "off"
    draftChunk:
      minChars: 200
      maxChars: 800
```
